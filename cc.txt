BoardInterface 类封装了主机与 FPGA 板卡之间通过 XDMA (PCIe DMA) 驱动的通信。它负责打开字符设备文件、分配 DMA 缓冲区、以及数据的收发。

发现的问题
1. xdma_send() 存在死循环风险 (严重)

while (count < size) {
    rc = write(fd, buf, size);  // 每次都写 size 字节
    assert(rc == size || rc == 0);
    count += rc;
}
write() 每次都传 size 而非 size - count，即便已经写了部分数据，仍然尝试从头写全部 size 字节。
如果 write() 永远返回 0（例如设备暂时不可写但未报错），count 永远不增长，循环永远不会退出。
应该使用 buf + count 和 size - count。
2. assert 在 Release 构建中会失效
代码中使用 assert 做了多处关键校验：


assert(rc == size || rc == 0);       // xdma_send
assert(size <= RECV_BUF_SIZE && "given read size is too large");  // xdma_recv
assert(count <= size);               // xdma_recv
assert 在 NDEBUG 模式下会被编译移除，这些校验将完全消失。对于硬件 I/O 操作，应该使用运行时错误处理（返回错误码或抛异常）而非 assert。

3. 资源泄漏 / 未初始化指针 (中等)
构造函数没有初始化 send_buf、recv_buf、to_card、from_card。如果构造后从未调用 init() 就析构，free() 和 close() 会操作未初始化的值，导致未定义行为。


BoardInterface::~BoardInterface()
{
    free(send_buf);     // 未初始化 — UB
    free(recv_buf);
    close(to_card);     // 未初始化 — UB
    close(from_card);
}
4. recvData() 返回值语义不一致
xdma_recv() 返回实际读取的字节数 count，但 sendData() / xdma_send() 返回 0 或 1（成功/失败）。调用方无法统一处理：发送侧看返回值是否为 0 判断成功，接收侧看返回值是否 > 0 判断成功。

5. break 语句永不可达 (轻微)

case IFACE::XDMA:
    return xdma_send(data,size);
    break;  // return 之后永远不会执行
return 之后的 break 是死代码，虽然无害但不整洁。

6. 头文件缺少 <cstdint> 包含
board.h 中使用了 uint（应该是 unsigned int），但没有包含 <cstdint> 或 <cstddef>。虽然可能通过其他头文件间接引入，但不是良好的可移植做法。

改进建议
xdma_send 的 while 循环应改为使用正确的偏移量和剩余字节数。
用运行时检查（if + 返回错误码）替换 assert。
构造函数中初始化指针为 nullptr，文件描述符为 -1；析构时检查后再释放。

【当前架构】

【目标架构】

核心变化：PCIe 链路被替换为 FPGA 内部的 AXI-Stream FIFO，x86 主机被替换为片上 MicroBlaze 软核。

一、可行性的四个层次
层次 1：AXI-Stream FIFO 替换 XDMA（中等可行）
XDMA 使用 /dev/xdma* 字符设备的 read()/write()。AXI-Stream FIFO 在 MicroBlaze 上通过 内存映射寄存器 访问：

XDMA 操作	AXI-Stream FIFO 对应
write(fd, buf, size)	循环写 AXI_FIFO_TX_DATA 寄存器，检查 TX_FULL 标志
read(fd, buf, size)	循环读 AXI_FIFO_RX_DATA 寄存器，检查 RX_EMPTY 标志
open("/dev/xdma*")	不需要 — FIFO 寄存器在 MicroBlaze 地址空间固定映射
posix_memalign	__attribute__((aligned(4096))) 静态缓冲区或 memalign()
32KB 短读检测	AXI-Stream 的 TLAST 信号或 FIFO 空标志
BoardInterface 的 switch/枚举设计就是为了支持这种替换 —— 直接加 IFACE::AXIS，实现对应的 axis_init/axis_send/axis_recv 即可。

层次 2：POSIX/Linux 依赖的剥离（较大改动）
经过全面扫描，以下 Blocker 级别的依赖必须处理：

依赖	所在文件	MicroBlaze 替代方案
open/close/read/write 设备文件	board.cpp	内存映射寄存器访问 (Xil_Out32/Xil_In32)
/dev/xdma* 路径	board.h	宏改为 XPAR_AXI_FIFO_* 基地址
std::thread 后台读线程	platform.h platform.cpp	直接去掉 — MicroBlaze 裸机上用同步轮询
boost::lockfree::spsc_queue	platform.h	去掉 — 不再需要线程间缓冲
fork/exec/mkfifo/wait (debug 功能)	prog.cpp	#ifdef 裁剪掉，裸机上不需要 Vivado 调试流程
system() 调用汇编器	prog.cpp	链接 smc_parser 为库直接调用
getpid() PRNG 种子	apps/*.cpp	替换为硬件定时器值或固定种子
gettimeofday 测量	retention.cpp	替换为 AXI Timer 读取

层次 3：FPGA HDL 的改造（最大工作量，关键路径）
当前 hdl/softmc_top.v 对接的是 XDMA IP 的 AXI-MM 从接口。XDMA IP 核收到 PCIe TLP 后，在 FPGA 侧暴露一个 AXI4-MM slave，指令被写入特定地址范围。

换成 MicroBlaze + AXI-Stream FIFO 后：

XDMA IP 核移除 — 不再需要 PCIe endpoint
添加 AXI-Stream FIFO IP (Xilinx axis_data_fifo 或 axi_fifo_mm_s)
softmc_top 的输入接口改为 AXI-Stream — 需要一个简单的包解析器来区分控制命令和指令数据
指令存储 — 可以用 BRAM 替代原来的寄存器，MicroBlaze 直接把指令序列写入 BRAM，然后 FPGA 执行引擎从 BRAM 读取
读数据返回 — FPGA 执行 DRAM READ 后，数据通过 AXI-Stream FIFO 返回给 MicroBlaze
这部分是 FPGA 工程师的工作，不在 C++ 源码范围内，但它是决定项目周期的关键因素。

层次 4：C++ 运行时在 MicroBlaze 上的适配
MicroBlaze 使用 Xilinx SDK/Vitis 工具链，基于 newlib 或 newlib-nano 标准 C 库，GCC 编译器。各特性的兼容性：

特性	可用性	备注
STL 容器 (vector/map/list/string)	可用	需要堆分配，增大 binary size
malloc/free/new/delete	可用	需配置 heap 大小（链接脚本）
<iostream> / printf	可用	需 UART 后端（semihosting 或 xil_printf）
<fstream> 文件读写	部分可用	需 FATFS 库 + SD 卡，或改用内存缓冲

assert()	可自定义	替换为 xil_printf + while(1)
posix_memalign	不可用	用 memalign() 或静态对齐数组替代
std::thread	不可用（裸机）	必须去掉，改为同步轮询模型
Boost 库	部分可用	SPSC queue 去掉后不再需要 Boost
二、具体改造方案建议
2.1 board.cpp 重构为 AXI-Stream 版本

// 伪代码 — MicroBlaze AXI-Stream FIFO 版本
#include "xil_io.h"        // Xil_Out32, Xil_In32
#include "xparameters.h"   // XPAR_* 地址宏

int BoardInterface::axis_init()
{
    // 1. 复位 AXI-Stream FIFO
    Xil_Out32(XPAR_AXIS_FIFO_TX_RESET, 1);
    Xil_Out32(XPAR_AXIS_FIFO_RX_RESET, 1);
    // 2. 分配静态缓冲区（不需要 posix_memalign）
    send_buf = (void*)malloc(SEND_BUF_SIZE);
    recv_buf = (void*)malloc(RECV_BUF_SIZE);
    // 3. 不再有文件描述符
    return 0;
}

int BoardInterface::axis_send(void* data, const uint size)
{
    uint32_t* words = (uint32_t*)data;
    int count = size / 4;
    for (int i = 0; i < count; i++) {
        // 等待 TX FIFO 不满
        while (Xil_In32(XPAR_AXIS_FIFO_TX_VACANCY) == 0);
        Xil_Out32(XPAR_AXIS_FIFO_TX_DATA, words[i]);
    }
    return 0;
}

int BoardInterface::axis_recv(void* buf, const uint size)
{
    uint32_t* words = (uint32_t*)buf;
    int count = size / 4;
    for (int i = 0; i < count; i++) {
        // 等待 RX FIFO 不空，或检测 TLAST
        while (Xil_In32(XPAR_AXIS_FIFO_RX_VACANCY) == FIFO_DEPTH);
        words[i] = Xil_In32(XPAR_AXIS_FIFO_RX_DATA);
    }
    return size;
}

#define BOARD_AXIS_H

/**
 * AXI-Stream FIFO Hardware Abstraction Layer for MicroBlaze.
 *
 * Maps the Xilinx AXI4-Stream FIFO (axi_fifo_mm_s) register interface:
 *   TX channel: TDFD (data write) at +0x00, TDFV (vacancy) at +0x04
 *   RX channel: RDFD (data read)  at +0x00, RDFO (occupancy) at +0x04
 *
 * Base addresses come from xparameters.h (generated by Vitis from the
 * hardware design). Override AXIS_TX_FIFO_BASEADDR / AXIS_RX_FIFO_BASEADDR
 * if a different mapping is needed.
 */

#include <stdint.h>
#include <string.h>

/* ---- FIFO register offsets (in bytes) ---- */
#define AXIS_FIFO_TDFD   0x00   /* Transmit Data FIFO Data          (write) */
#define AXIS_FIFO_TDFV   0x04   /* Transmit Data FIFO Vacancy       (read)  */
#define AXIS_FIFO_RDFD   0x00   /* Receive  Data FIFO Data          (read)  */
#define AXIS_FIFO_RDFO   0x04   /* Receive  Data FIFO Occupancy     (read)  */
#define AXIS_FIFO_SRR    0x10   /* Software Reset Register                   */
#define AXIS_FIFO_TDFR   0x00   /* Transmit Data FIFO Reset  (write 1 to SRR) */

/* ---- Default base addresses (override via -D or xparameters.h) ---- */
#ifndef AXIS_TX_FIFO_BASEADDR
#ifdef XPAR_AXI_FIFO_MM_S_0_TX_BASEADDR
#define AXIS_TX_FIFO_BASEADDR  XPAR_AXI_FIFO_MM_S_0_TX_BASEADDR
#else
#define AXIS_TX_FIFO_BASEADDR  0x44A00000
#endif
#endif

#ifndef AXIS_RX_FIFO_BASEADDR
#ifdef XPAR_AXI_FIFO_MM_S_0_RX_BASEADDR
#define AXIS_RX_FIFO_BASEADDR  XPAR_AXI_FIFO_MM_S_0_RX_BASEADDR
#else
#define AXIS_RX_FIFO_BASEADDR  0x44A10000
#endif
#endif

/* ---- FIFO status queries ---- */
static inline uint32_t axis_tx_vacancy(uintptr_t base) {
    return axis_reg_read(base + AXIS_FIFO_TDFV);
}

static inline uint32_t axis_rx_occupancy(uintptr_t base) {
    return axis_reg_read(base + AXIS_FIFO_RDFO);
}

/* ---- FIFO data I/O ---- */
static inline void axis_fifo_put(uintptr_t base, uint32_t word) {
    axis_reg_write(base + AXIS_FIFO_TDFD, word);
}

static inline uint32_t axis_fifo_get(uintptr_t base) {
    return axis_reg_read(base + AXIS_FIFO_RDFD);
}

/* ---- Blocking send / recv of byte buffers over AXI-Stream FIFOs ---- */

/**
 * Write `size` bytes from `data` to the TX FIFO in 32-bit words.
 * Blocks while FIFO is full.  `size` should be a multiple of 4.
 * Returns 0 on success, non-zero on timeout (if implemented).
 */
static inline int axis_fifo_send(uintptr_t tx_base, const void* data, unsigned int size) {
    const uint32_t* words = (const uint32_t*)data;
    unsigned int count = size / 4;

    for (unsigned int i = 0; i < count; i++) {
        /* spin while FIFO full (vacancy == 0) */
        while (axis_tx_vacancy(tx_base) == 0) { }
        axis_fifo_put(tx_base, words[i]);
    }

    /* trailing bytes (if size not word-aligned) — pad to word */
    unsigned int rem = size % 4;
    if (rem) {
        uint32_t last = 0;
        memcpy(&last, (const uint8_t*)data + (count * 4), rem);
        while (axis_tx_vacancy(tx_base) == 0) { }
        axis_fifo_put(tx_base, last);
    }
    return 0;
}

/**
 * Read `size` bytes from the RX FIFO into `buf` in 32-bit words.
 * Blocks while FIFO is empty.  `size` should be a multiple of 4.
 * Returns the number of bytes actually read.
 */
static inline int axis_fifo_recv(uintptr_t rx_base, void* buf, unsigned int size) {
    uint32_t* words = (uint32_t*)buf;
    unsigned int count = size / 4;

    for (unsigned int i = 0; i < count; i++) {
        /* spin while FIFO empty (occupancy == 0) */
        while (axis_rx_occupancy(rx_base) == 0) { }
        words[i] = axis_fifo_get(rx_base);
    }

    /* trailing bytes — read the word but only copy the needed bytes */
    unsigned int rem = size % 4;
    if (rem) {
        while (axis_rx_occupancy(rx_base) == 0) { }
        uint32_t last = axis_fifo_get(rx_base);
        memcpy((uint8_t*)buf + (count * 4), &last, rem);
    }
    return (int)size;
}

int BoardInterface::sendData(void* data, const unsigned int size)
{
  const unsigned int count = size / 4;

  for (unsigned int i = 0; i < count; i++)
    putfsl(((unsigned int*)data)[i], FSL_TX_SLOT);

  /* trailing bytes — pad to word */
  unsigned int rem = size % 4;

  const unsigned int count = size / 4;

  for (unsigned int i = 0; i < count; i++)
    getfsl(((unsigned int*)buf)[i], FSL_RX_SLOT);

  /* trailing bytes — read full word, copy what's needed */
  unsigned int rem = size % 4;
  if (rem) {
    unsigned int last;
    getfsl(last, FSL_RX_SLOT);
    memcpy((unsigned char*)buf + count * 4, &last, rem);
  }

  return size;
}

putfsl/getfsl 本质是宏，不是函数。 在 Xilinx <fsl.h> 中它们是这样定义的：


#define putfsl(val, id)  __asm__ __volatile__ ("put  %0, rfsl" #id :: "r"(val))
#define getfsl(val, id)  __asm__ __volatile__ ("get  %0, rfsl" #id : "=r"(val))
宏由预处理器展开，不受 C++ 名字改编影响。所以原来的写法在大多数情况下实际能编译通过。

但 <fsl.h> 内部可能还声明了其他辅助函数或内联函数，没有 extern "C" 保护的话会有链接问题。加 extern "C" 是 C++ 调用 C 头文件的标准防御性写法，确保万无一失。

另外，如果某个 MicroBlaze 工具链没有 <fsl.h>（某些版本只用编译器 builtin），可以改用内置函数，无需头文件：


#define fsl_put(val, id)  __builtin_microblaze_putfsl((val), (id))
#define fsl_get(val, id)  __builtin_microblaze_getfsl((val), (id))

cout (C++) 和 printf (C) 的兼容性
二者在裸机上走的是同一条路——都需要你提供 _write() 桩函数。


std::cout << "hello"   ──┐
                         ├──→ newlib write() ──→ _write(fd, buf, len) ──→ ???
printf("hello")        ──┘
MicroBlaze 裸机上，_write() 默认是一个空桩（返回 -1），所以 cout 和 printf 都能编译通过，但输出会被丢弃，不会打印到任何地方。

要看到输出，需要在工程里实现 _write()：


// 添加到 Vitis 工程中
#include <xil_io.h>

int _write(int fd, const char* buf, int len) {
    for (int i = 0; i < len; i++) {
        // 轮询等待 UART TX FIFO 有空位
        while (Xil_In32(UART_BASEADDR + 0x2C) & 0x08);
        // 发送一个字节
        Xil_Out32(UART_BASEADDR + 0x30, buf[i]);
    }
    return len;
}
Vitis 的 helloworld 模板通常自带这个桩（在 syscall.c 或 BSP 中）。如果你的工程里已经有它，那 cout 和 printf 都能正常工作，不需要做任何区分。
